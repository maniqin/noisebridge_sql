What is SQL and why is it used?

  • Strucured Query Language
    • SQL (pronounced 'es-que-el' or 'sequel')
  • We access databases with it
  • Fast / efficient storage & retrieval
  • One of the most used / popular ways to store data

What is SQL and why is it used?

  • Key component to backends of many Web Apps
  • SQL is often abstracted with ORMs (Object-relational mapping)
    • Nonetheless, still widely used in its raw form for data warehousing, analytics, ML, apps optimization...

Key DB concepts

  • Different types of Databases optimized to do different things
    • Concept of analytical db vs operational db

  • DB on file (sqlite) vs DB on server (e.g. PostgreSQL)

  • Understanding why it's called RDBMS (relational database management system)
    • Can decide what are the relationships between tables before inserting the data
    • Very important step, if not the most important, may have repercussions in the future

Tables

  • What is a table?
    • Wikipedia Ref
  • Why multiple tables and not just one?
    • Storage efficiency
    • Data better organized
    • Encapsulate "concepts" into tables

What is a Column or Field?

  • A field is a column in a table
  • Vertical entity that stores data
  • We can apply filters on columns
    • just like you would on a spreadsheet
  • Columns store a specific data types
    • more on this later

What is a table schema?

  • A schema is a collection of (most likely) related tables
  • How do I draw / design one?

Understanding Constrants

  • When creating a table we can set specific constraints to it
  • This means columns we specify will follow set rules (e.g. never being empty)
  • Some examples of constraints: NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY, etc..
  • w3 schools ref

Data Types

  • Columns are created with a set data type
    • This means that a given column will accomodate a specific type of data and refuse other ones

  • Most common data types:
    • integers
    • floats
    • date/datetime/timestamp
    • varchar
    • boolean
    • autoincrement/sequence

Let's write some sql

SELECT statement

  • SELECT, as the word implies, "selects" data from tables
  • SQL way to retrieve data from one or multiple tables
  • May be the most common statement you'll see around
  • 99.9% of the times it will be followed by a FROM
  • Doesn't matter if typed upper or lower case
  • Indentation and spacing in SQL doesn't matter

SELECT from a table

  • All columns
    • 							
									SELECT * FROM table_name
  • Specific columns, choose the order you want
    • 							
									SELECT  
									column_name_1,
									column_name_2,
									column_name_3
									FROM table

What if the table outputs too many rows?

Use limit!
							
									SELECT 
									column_1,
									column_2 
									FROM table_name
									LIMIT 100;
